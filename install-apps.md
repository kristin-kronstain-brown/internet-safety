# "Can I install this?"

There are always going to be new apps that kids want to install. Form a plan for deciding which apps are OK to install and which are not.

## Preparing a device

Before you start installing apps, lay the foundation for the future. Even if you are just giving your 3 year old a tablet to watch in the car, take these steps early. That way, when they're 8, they will be used to the process.

1. Set up the device using a family Apple/Gmail account.

    - Can't use the same login for multiple games though.
    - Use a generic name and random photo.

1. Install a parent monitoring app.

1. Learn what settings you do have control over in the parent monitoring app.




## To install or not to install?

Go through this list of questions and decide what you are OK with for the age your child is. Repeat this process and adjust your answers as the kids get older.

### What is the age rating?

Ratings might not be directly related to what you care about as parents. A game can be rated **Everyone** or **4+** and still have an open chat. But it's a place to start. If the app is rated for teens, then it can be a quick no for an 8 year old.

Learn more about the age-based ratings for your operating system:
- [Google Play](https://support.google.com/googleplay/answer/6209544)
    - In any app's details page, in the Information section, in addition to the age rating, look for tags or labels. 
- [Apple](https://developer.apple.com/help/app-store-connect/reference/app-information/age-ratings-values-and-definitions/)
    - In any app's details page, in the Information section, look for "Age rating" and "Contains" sections. 

### Could the permissions required legitmiately be used to make the software better?
There is an expected amount of permissions an app might require. These things could be gathered to make the software better, rather than collect information about you personally.


#### Generally reasonable

These are common permissions that can genuinely improve an app without being suspicious by themselves:

- Internet access
- Notifications
- Storage/photos access to store output from the app
- Approximate location
- Name, profile picture, email address
- Device manufacturer information

#### Requires stronger justification

These permissions aren't necessarily bad, but users should expect a clear explanation:

- Camera
- Microphone
- Storage/photos access
- Precise location
- Contacts
- Calendar
- Bluetooth
- Health/Fitness data
- SMS access
- Call logs
- Accessibility services
- Device administrator privileges

#### Potential red flags

For many games and simple apps, you should ask why these are needed:

- In-app browser instead of using the system default browser
- Contacts access
- SMS reading
- Call history access
- Continuous background location
- Accessibility permissions
- Permission to install other apps
- Broad file-system access
